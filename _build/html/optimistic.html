Optimistic Solver¶

class libdoug.optimistic_solver.OptimisticConflictSolver(allimages, delta, repo, rt=<class 'libdoug.conflict_resolution.Resolution'>)[source]¶

This sublcass of libdoug.conflict.ConflictSolver class is called optimistic because it works as follows:

  1. Push what’s missing in the remote
  2. Pull what’s missing locally

Conflicts are solved by syncing both sides to be the same, it could also be called naive.

Note

libdoug also bundles other, more intricate solvers, such as the libdoug.vr_solver.VRConflictSolver
